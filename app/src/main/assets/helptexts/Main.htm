A Japanese style Mahjong game to play among your friends. No specific server machine is required.
You can also play among less than 4 member or play alone with 3 Robots.
I suppose you are familiar with the rules and operation of Japanese style Mahjong. You may be required to become familiar with some Kanji(漢字) glyph a bit.
Evaluation of Value of Hands is optional(the default is in "Setting,Operation" dialog. "Dialog" means panels popup to accept your response.）
If you do not agree the score called, you can adjust the score.
Depending on setting of "Check winning hand", 錯和("Chombo") by mistaken "Win" call may occur.
Win anyway Menu-Item allows any irregular local Yaku(=役=winning pattern).
Rule of 焼き鳥(=Yakitori), 割目(=Yareme) and 飛(=Tobi) is optional (See Help of RuleSettings dialog about those terms).
Suspend and resume is supported.
Device is required portrait width 600 pixels over and 800 pixels is recommended.
From Android-6.0(Marshmallow), you will be asked to allow the permission of "Write to external storage"(Write to external storage). Give the permission for saving Rule files and History-Record files. Without permission, those are saved to internal storage which are cleared at re-installation. If you denied with "Don't ask again", re-installation required to allow the permission again.
From Android-11, you will be asked to specify the folder in external storage at first startup after (re-)installation. Select folder:eMahjong. If it not exist, make the folder using the dialog. Old data will be migrated to the folder. "Cancel":ask at restart, "Fix=NO":ask again after reinstall. If not granted, saved to internal folder which cleared at re-installation.
Additinally you will be asked to allow "READ permission" to listup audio file for BGM.
And permission:ACCESS_FINE_LOCTION is required to use WiFi-Direct. Allow it.

For Lang=ko environment, help text by Hungle is obtained by Google translation(https:translate.google.com)

Followings are outline to start game.
Select one server device out of 4 devices, all clients Connect to the server. Server setups Rule then send to all clients. After all clients agreed to the rule, Server pushes Game to switch to game panel.
On the game screen, until the first dealer's Draw, the procedure of deciding a temporary parent, deciding seat and dealing tiles have to be done. The procedure proceeds by touching the dice or semicircular lamp in the center of board. When your device shines, you have to touch it.
For details see Help on each dialog.
To start play-alone push "Game" button without connection.
----------------------------------------------------------------------------
Connect : establish connection among players.
Someone has to become Server, the others(called Client）connect to the server.
When connected, button background changes and server is Gold color.
Wireless can reach about 50M, I suppose to play in the distance of analog conversation available, which may be frequently required. However, due to various factors, keep social distance.
If connection is cut off by telephone call, you may have to set "Aeroplane mode" in Android "Network & Internet" setting.

Settings：RuleSettings and Preference.
RuleSettings：Set "Rounds", "PlacingPrize", "Yakitori", "Wareme", "Tobi",...
Once at after installation, before starting game, It is required to open Rule setting dialog, update and push OK.
By child dialog, set rule related to Yaku and Operation.
After rule was synchronized among all players,Game can be start. At first time of connection and after rule was changed Warning "Rule unmatch" will be issued. Synchronization may fail by the reason of APP version unmatch. Open Settings dialog and Send it to all clients to obtain O.K.'ing. Rule files can be saved/reloaded, select a file suitable for the group each.
Game advances by claiming and O.K. by dialogs sent and received at each scene. The player other than claimer may send response "NG". The dealer of the hand has authority of final decision regardless of "NG" response.
e.g. On "End-of-Hand", dealer can set "Chombo" or "Invalid" to the win claimed and can force dealer's decision regardless "NG" response. On some other dialogs, server or dealer at starting game has the authority.
Some items of rule are not checked by the App and are merely promises, those will be desided after mutual consultation.
Preference : Screen orientation, Sound etc.

Game：Server only can start game after rule synchronization was done successfully. When number of players connected < 4, You can optionally start game with Robot Human players owe Robot's defeat by rule setting.

Records : shows List of game results.
Suspended game is resumed from this List. See SuspendOption of RuleSetting->Operations,Help for how to ssuspend game.
You can show Balance Sheet of all games, or specific member's game from records.

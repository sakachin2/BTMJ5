Japanese style Mahjong game to play among your friends. No specific server machine is required.
I suppose you are familiar to rule and operation of Japanese style Mahjong. (You may be required to become familiar with some Kanji glyph a bit.)
Any local Yaku(winning pattern) is acceptable because winner can claim the value of winning hand. Depending on setting of "Check winning hand", Chombo by mistaken Ron may occur. Rule of Yakitori, Wareme and Tobi is optional (See Help of RuleSettings dialog about those terms).
From 2 to 4 players can play game. Suspend and resume is supported.
Device is required portrait width 600 pixels over and 800 pixels is recommended.
From Android6.0(Marshmallow), you will be asked to allow the permission of "Write to external storage". Give the permission for saving Rule file and Game-Record file.

Connect : establish connection to the player.
Someone have to become Server, the others(called Client）connect to the server. Wireless can reach about 50m, I suppose to play in the distance of analog conversation available, which may be frequently required. However, due to various factors, keep distance of 6 feet.

Settings：RuleSettings and Preference.
RuleSettings：Set "Rounds" of game, "PlacingPrize", "Yakitori", "Wareme", "Tobi",...
By child dialog, set rule related to Yaku and Operation. (dialog means panel popup to accept player's response.）
After rule was synchronized among all players,Game can be start. If rule was changed, "Send" it to all clients to get agreements. Rule files can be saved/reloaded, select a file suitable for the group each.
Game advances by claiming and agreeing by dialogs sent and received at each scene. Other player than claimer may send response "NG". The dealer of the hand has autority of final decision regardless of "NG" response.
e.g. On "End-of-Hand", dealer can set "Chombo" or "Invalid" to the win claimed and can force dealer's decision regardless "NG" response. Exceptionally on some dialog, server or dealer at starting game has the autority.
Preference : Screen orientation, Sound etc.

Game：Server only can start game after rule synchronization was done successfully. When number of players connected < 4, You can optionally start game with Robot(Dead Hand:just discard drawn tile). Human players owe Robot's defeat by rule setting.

Records : shows List of game results.
Suspended game is resumed from this List. To suspend game, select "Rquest suspend" on menu during game. When connection failure occured game can be suspended also. The former, "SuspendGame Claim" dialog is shown at end of the hand, and SuspendGame or Gameover are selectable. The latter, you can try "Reconnect" and continue the hand if recovery succeded. Otherwide SuspendGame or Gameover are selectable as the current hand was lost.
Note. 　"hand" is used in two way.
　　1. tiles in hand.　　2. end of hand:anyone won in the hand(shot).
　"draw" is used in two way.
　　1. draw tile:pick up a tile from wall.　　2. drawn hand:shot in which no one won.

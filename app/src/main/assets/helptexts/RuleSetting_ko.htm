Yaku 버튼 : "Kuitan"(喰-断么九), "Yakuman"(役満)과 같은 Yaku(役:승리 패턴)와 관련된 하위 대화 상자를 엽니다.
Oper 버튼 : "DelayTime", "Check win hand"와 같은 작업과 관련된 하위 대화 상자를 엽니다.

ID : 규칙 파일을 식별합니다. 간사이(関西), 도쿄(東京) 등으로 채우고, 플레이어 그룹에 따라 규칙 파일을 다르게 사용합니다.
업데이트 시 Save/Load 버튼을 사용하세요.
ID 오른쪽에는 Update(업데이트)의 타임스탬프가 표시됩니다. 이것이 규칙 동기화의 핵심입니다. 플레이어 간의 불일치는 내용이 같더라도 게임 시작을 방해합니다. 규칙이 업데이트되면 서버는 모든 클라이언트에 Send해야 합니다. 클라이언트가 수신되면 이 대화 상자가 열립니다. 배경색으로 현재 설정과 다른 항목을 식별할 수 있습니다. 모든 클라이언트로부터 Agree를 받으면 게임을 시작할 수 있습니다. 클라이언트에서 규칙 파일을 사용하려면 클라이언트가 파일을 "Load"한 다음 서버로 "Send"하십시오. 동기화를 위해 다른 클라이언트로 보내야 하는 서버에서 대화 상자가 열립니다.
OK : 업데이트가 "Current"로 저장되고 활성화된 상태로 유지됩니다.
업데이트는 "Send" 버튼으로도 저장됩니다. "Current"는 내부 폴더에 보관되며 재설치 시 제거됩니다.
Cancel : 업데이트가 삭제됩니다.

Rounds:

東東戦:동 전체 8라운드.
東南戦: 東이 4발, 南이 4발.
東北戦: 東으로 4발, 北으로 4발.

InitialScore : TopPrize(최고 플레이어의 추가 이득)=(30000-InitialScore)*4
Noten:3000: Exaustive Draw(流局)에서 Noten(NO-聴牌)의 패널티. Noten 플레이어는 총 3000을 지불하고 Tenpai(聴牌) 플레이어는 3000을 얻습니다. 예를 들어 1 노튼은 3000을 지불하고 3 텐파이는 각각 1000을 얻습니다.
Counter: 카운터 스틱 값. 딜러가 딜러(連荘)로 계속 남아있을 때 카운터 스틱(들)이 보드에 표시됩니다. 승자는 (카운터 스틱의 가치)*(카운터 스틱의 수)의 추가 점수를 얻습니다.

PlacePrize : Placing(주문)에 의한 지불.
예를 들어 5-10일 때 마지막 플레이어는 상위 플레이어에게 10,000을 지불하고 세 번째 플레이어는 두 번째 플레이어에게 5,000를 지불합니다.
HeadBump: 시계 반대 방향 순서로 시작 딜러에 가장 가까운 플레이어가 더 높은 Placing(order) 플레이어를 얻습니다. Divide(cut100):100으로 내림합니다.

Fraction : 100보다 작은 반올림 포인트.
음수는 다음과 같이 반올림됩니다.
4Down 5Up : -100 ~ -500 ===> 0,　-600 ~ -900 ===> -1
5Down 6Up : -100 ~ -400 ===> 0,　-500 ~ -900 ===> -1
Round-Up　: -100 ~ -900 ===> 0
Round-Down: -100 ~ -900 ===> 0
상위 플레이어의 최종 점수는 제로섬을 유지하도록 조정됩니다.

Dora: Dora 타일은 각각 1 han(翻:Rank)의 값을 가집니다.
Dora는 무작위로 결정되어 매 라운드 시작 시 Stock에 표시되며, Kan(槓:declare quad)이 선언되면 추가 Dora가 열립니다.
누군가 Chankan을 선언하면 Kan(槓)-Dora가 열리지 않습니다. Chankan(槍槓): Add-Kan(加槓)의 타일에 대해 승리를 선언합니다.
Hidden : Stock에서 열린 Dora 타일 아래에 있는 타일입니다. 이미 Riichi(立直)를 선언한 승자에게 유효합니다.
Itself ​​: Dora는 열린 타일과 같은 타일입니다.
NextTo : Dora는 열린 타일의 다음 타일입니다.数牌:1-9-->2-0, 三元牌:WGR(白,發,中)-->GRW(發,中,白), 風牌:ESWN(東,南.西,北)-->SWNE(南,西,北,東)
Imm.는 王牌의 Draw 직후에 Kan(槓)-Dora가 열리는 것을 의미합니다. ON으로 설정하지 않으면 Discard 이후에 Dora가 열립니다.
RedDora：(赤도라) 5萬,5筒,5索의 2/4는 붉은색이다. 총 3*2=6 타일은 Dora로 간주됩니다.

Wareme(割目)：
Wareme(일본어:fissure) 플레이어는 각 라운드가 시작될 때 던진 딜러의 역할에 따라 결정됩니다.
주사위 반점=2-->南은 Warame, 3-->西, 4-->北, 5-->東, 6-->南, ...

Tobi :(飛び) (일본어: Fly away, Gameover by 파산)
Pay for negative score: 느슨한 쪽이 승점을 지불합니다. 결과적으로 루저 점수는 마이너스가 될 수 있습니다.
Return after placecing: 플레이어는 모든 승리 포인트를 한 번 지불합니다. 최종 점수로 선수의 순위가 결정된 후, 승자는 패자의 최종 점수가 0이 될 때까지 패자에게 상환합니다.
Limit winner's point : 승자의 포인트는 루저의 현재 점수로 제한됩니다. 결과적으로 루저의 점수는 0이 됩니다. 플레이어의 위치는 최종 점수에 의해 결정됩니다.
Double-Ron:Yes인 경우 루저가 모든 승자에게 지불합니다. 패자에 가장 가까운 승자부터 시작하여 시계 반대 방향 순서에 따라. Tobi는 해당 시퀀스에서 파산이 발생했을 때 플레이어에게 주어집니다.
NagashiMangan(流し満貫)일 때는 현재의 부모(東), 혹은 부모에 가장 가까운 곳에서 징수를 시작한다.

Yakitori：(焼き鳥, 일본어:YAKI(焼:그릴)-TORI(鳥:새, 치킨)
이 표시는 플레이어가 이 게임에서 한 번도 이기지 못했음을 나타냅니다. 게임 종료 시 마크를 들고 있는 플레이어가 페널티를 지불합니다. 이 패널티는 플레이어의 순위가 결정된 후에 지불됩니다.
Divide: 표시된 플레이어가 패널티를 지불합니다. 페널티 포인트는 표시되지 않은 플레이어에게 균등하게 분배됩니다.
To All: 표시된 플레이어는 표시되지 않은 모든 플레이어에게 패널티를 지불합니다.

Call the same Meld： Pon(碰) 또는 Chii(吃) 호출 후 폐기 제한.
No if the same tile: 예) 1번 Chii를 호출하여 멜드(順子) 1-2-3을 만들고 1번을 버리는 것은 허용되지 않습니다. 그러나 4번을 버리는 것은 허용됩니다. 마찬가지로 Pon을 호출한 후 동일한 타일을 버리는 것은 허용되지 않습니다.
No: 둘 다 허용되지 않습니다.
OK fully: 둘 다 허용됩니다.

Abortive Draw(流局)：
4Riichi：(四家立直:모든 플레이어는 Riichi라고 불림)
버린 타일에 대해 아무도 Ron(栄和)를 호출하지 않으면 몇 초가 지나면 대화 상자가 자동으로 팝업됩니다.
4Winds：(四風子連打: 처음 버린 4개의 타일은 모두 동일하며 東,南,西,北 중 하나)
대화 상자가 자동으로 팝업됩니다. 暗槓(conceiled Kan)가 첫 번째 4 Discard에서 선언된 경우 4Winds 의 경우가 아닙니다. 그러나 이 경우 메뉴에서 대화 상자 열기를 요청할 수 있습니다.
3Win：(三家和:3명의 플레이어가 Win for the Tiles Discarded)를 선언했습니다.
모든 승자가 "End-Of-Hand" 대화 상자에서 "Fix"를 눌렀을 때 "AbortiveDraw" 대화 상자가 팝업됩니다. "Double-Ron"이 "No"인 경우에도 유효합니다.

Double-Ron：
No:HeadBump:시계 반대 방향으로 루저에 가장 가까운 플레이어에게만 승리가 부여됩니다.
Yes여러 플레이어의 승리를 허용합니다.積み棒(Counter stick)에 의한 점수는 HeadBump 규칙에 따릅니다. Yes also...Counter-Stick의 포인트도 모든 우승자에게 지급됩니다.
로봇의 Ron은 인간 플레이어가 혼자 모드로 플레이할 때를 제외하고 빨간색 기간(Ron 간격)에 Ron을 호출하면 이 옵션과 상관없이 무시됩니다.

Robot:
로봇의 이름은 "S それがし", "A あきらくん", "T てるちゃん", "M まあちゃん" 및 "K けんちゃん" 중에서 무작위로 선택됩니다.
일본어 발음은 Soregasi(나 자신을 의미), AKIRA-kun, TERU-chan, MAH-chan 및 KEN-chan입니다. 그들은 내 어린 시절 친구입니다.
스마트 로봇은 노란색 기간(Pon(碰)/Kan(槓) 간격)에 들어가기 직전에 Ron을 발행하고 존재하는 경우 인간 승자에게 양보합니다. 하지만 혼자 플레이 모드에서는 인간 플레이어가 우선권이 없으며 Double-Ron 규칙이 적용됩니다.
혼자 플레이하는 경우를 제외하고 게임에 로봇이 포함된 경우 로봇의 부채는 플레이어에게 책임이 있습니다.
Divide ​​: 로봇의 총 부채를 균등하게 나눕니다. 분수는 1위 플레이어의 최종 점수로 제로섬을 조정하고 있습니다.
In proportion by score : 가장 낮은 순위의 인간 플레이어의 점수가 0이 되도록 비례적으로 나눕니다.
    예: 점수: +60000(상단), +10000, -30000(마지막), -40000(로봇)
       6-(-3)==>11, 1-(-3)==>4, -3-(-3)==>0  합계 =15가 분할 대상입니다.
       11/15*(-4)=-29300(100으로 반올림), 4/15*(-4)=-10600, 0/15*(-4 )=0
       제로섬 조정 -29400, -10600, 0, +40000이 점수에 추가됨
       드디어 +30600, -400, -30000, 0(로봇)

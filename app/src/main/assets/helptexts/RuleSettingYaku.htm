Rules about Yaku(define pattern of winning hand and its value).
To check whether winning hand is complete or not, set "Do check" on topic 1-han constraint of Oper Setting. Even if the check is set on, "Win" can be claimed for any hand by menu:Win anyway or Win-A button.
After you called "Win", open Win Claim dialog by Score button. Then claim your hand's value by count of han and fu. (You may set ON to Evaluate Point by APP of Oper settings.)
"Win Claim" is sent to the other players, after all player responded OK or NG, End-of-Hand dialog opens on the dealer which displays winning tiles.
If the dealer is Robot, the dialog opens on the next(counter clockwise) human player. The dealer checks validity of the win call, then send the dialog to the other player. After receiving "OK" from the other players, Fix the game to complete "End-of-Hand".
If dealer found "Chombo"("Furiten", "Noten", "Noten Riichi", "Constraint by han" etc), dealer can set Invalid(without penalty) or Chombo(with penalty). In that case dealer can select "Gameover" or "Continue" with "DeadHand" of "Chombo" players. DeadHand player continues to play just discarding drawn tile and can not do any call.

Psuedo-Tenpai
By Japanese "Keishiki-Tenpai", Keishiki means formal, only in form or structure.
In ExhaustiveDraw, if your-hand is not status of Tenpai with at least 1-han Yaku, you have to pay penalty of Noten(Not Tenpai). If dealer is Tenpai status, he can repeat next round as the dealer. This paragraph loosen the standard of Tenpai.
Disable Psuedo-Tenpai:
You have to be able to call win by Ron(not by self-draw). It is not recognize as Tenpai for the case of Furiten, Kataagari, Fix Yaku violation and 1/2-han constraint violation. Missing winning tile after the last Discard is also checked as Furiten. Empty winning tile remains is OK.
Furiten Riichi is not a Tenpai because it can be win only by Self-Draw.
If this setting is OFF, Psuedo-Tenpai is allowed as followings assuming you have self-drawn a winning tile.
0-han OK:No Yaku is required by any of winning tile. If OFF, at least 1-han except Dora is required by any one of winning tiles. Accordingly menzen(All-In-Hand) is recognized as Psuedo-Tenpay by 1-han of Self-Draw. Furiten is all OK.
Furiten Riichi may impose Chombo penalty in some case, bit it is recognized as Psuedo-Tenpai.
When 2-han constraint
2-han constraint status occurs by 2-han constraint setting bellow.
2-han required:One of winning tiles have to construct at least 2-han.
Riich+Self-Draw is 2-han by itself. You can not call win under 2-han constraint status because Self-Draw is a accidental Yaku. However it is recognized as Psuedo-Tenpai.

Allow OpenRiichi:1-han is added instead of exposing you hand(other player understands winning tiles.) NagashiMangan
Special rule to gain points at ExhaustiveDraw to the player that all Discards were 1, 9 and word tile, and any Discards were not called by other player. The player's call is out of question.
No:This rule is not applied. Win:Regard as Win. No Noten penalty, remove Yakitori mark, stay dealer if dealer win. Drawn:Regard as Drawn. Pay Noten penalty, leave Riichi beds, no additional CounterStick payment, rotate dealer if dealer is Noten.
In both case Multiple Ron setting is applied. HeadBump means nearest to East wind position. When Tobi occurred, Tobi prize player is determined by the paying point of NagashiMangan, then Noten penalty. Each is collected from the dealer or nearest position to the dealer. AbortiveDraw by "3 Win" rule is not applied.

8 consecutive win
According to the following definitions, the dealers's win becomes Yakuman.
However, in the operation settings, if "Evaluate Point" is OFF, or if "Check 1-han constraint" is OFF, "8 consecutive win" will be disabled. Local Yaku that are not standard winning pattern are not taken into consideration, so please set Yakuman in the "Win Claim" dialog. Even if setting of "NagashiMangan" is "Win", "NagashiMangan" resets the number of consecutive win and it will not be a winning of "8 consecutive win".
The number of consecutive wins for the player is displayed near the Dealer-Mark.
8 times as dealer: When "Number of consecutive wins > Number of piled sticks", the number of wins includes the win as the immediately preceding non-dealer. When "Number of consecutive wins < Number of piles sticks", the number of sticks include the count of Drawn-round. This option makes the dealer's win as Yakuman of "8 consecutive win" when the smaller one is 7.
9 times as dealer: When the Number of consecutive wins and the Number of piled sticks, whichever is smaller, is 8, the dealer's win will be considered as Yarkuman.
Start may be non-dealer: When the number of consecutive wins is 7, the dealer's win is considered as Yakuman.
Dealer's win when 8 sticks: Any number of stickes by Drawn-round can be included. This option is easy to judge.
Next is +8: If this is OFF, all dealer's win is Yakuman until the non-dealer's win or Drawn-round. If this is ON, the next 8 consecutive is the next Yakuman timing.

Operation related rules.

DelayTime
Set time period after Discard for "Ron", "Pon/Kan", "Chii/Draw". Semicircle lamp on center of board lights as the sequence of Red, Yellow, Green then Blue.

Ron : Red lamp. Only Ron is accepted. "Win" call out of this period is, if not first winner, "Late Ron" and is not accepted regardless "Double-Ron" rule. Accordingly, "3Win" is accepted as AbortiveDraw if 3 players issued "Win" in this period.
To accept Chankan Ron(Robbing a Quad) for added Kan, you can take Rinshan after this period + next "Pon/Kan" period. Likewise, 2nd win call is "Late Ron" when this time exhausted. If claming win to Ankan is allowed for Yakuman:Kokusi(13 Orphans) by settings, this period applied.
Pon/Kan : Yellow lamp. Ron(only first claimer)/Pon/Kan are accepted. When this time exhausted, lamp changes to Green and Ron(only first claimer)/Chii/Draw/Pon/Kan become available and the first comes. Lamp changes to Blue when the tile was drawn.
After Last Discard : After the last discard, "ExhaustiveDraw" completes after this time exhausted after lamp changed to green.
AutoDraw : From entered to green(draw-able) period, if this time exhausted next player draw automatically. Also applied to drawing Kan tile. 0 means no AutoDraw.
AutoDiscard : After (Kan)Draw or after Pon/Chii, tile framed green in hand is discarded automatically when this time exhausted. If set, you may miss to claim 9-9Tile, 13 broken, 14 broken etc. 0 means no AutoDiscard.
Notify availability of Ron,Pon,Kan,Chii.
In Match mode(Not Play Alone mode), notify availability of those by button color which back to normal state by timeout or other players action.
For Kan, not notify at Drawn a Tile but at other player discarded.
Notify availability of Ron,Pon,Kan,Chii when playing alone.
If set ON, DelayTime except Pon/Kan is ignored and Human player draws automatically in playing alone mode. It speed up playing time.
For Kan, not notify at Drawn a Tile but at other player discarded.

"Win" is not notified in case of Furiten, Multiple Wait one of which is 1/2 han constraint is NG and violation of Fix Yaku at First. And Furiten-Riichi-OK option is effective. 2 han constraint error at Riichi is notified if "Confirm Tenpai" option is ON.

When playing alone, Robot's "Draw" is by button
Robot's "Draw" is not by timeout but by button touch. Set on if you want to take a time hand by hand in playing alone mode.

Wait!
Win call is cancelable : If checked ON, you can block other player's action to take a time to consider the winning is better or worth for your score placement. In Wait! state, small yellow-green lamp lights notifying blocked status to the other players. Other player's Pon/Kan/Chii/Ron is queued, next player's Draw and AutoDraw are rejected. When pressed "Win" again, Win claim complete.
(This setting is ignored when playing alone with notify mode and win call is cancelable by "Cancel" button notified.)
Win call is canceled when "Cancel" pressed or timeout, if set, exhausted. Blocking is released and queued other player's actions are executed sequentially. AutoTake timer start if set.
Pon/Chii may cause Wait! state. That is, for Pon, Red-Dora is to be included or not, for Chii, multiple candidate of Melds exists. Selecting tile(s) in hand press again Pon/Chii, then the action completes. You can not cancel your Pon/Chii.
Timeout:Yes : When this time exhausted, The action is canceled and blocking is released. Queued action, if exist, is executed.
e.g.) In green period all action is allowed, anyone called Chii at first and entered to "Wait!" state by Melds of multiple candidate. At this time, if another player called Pon, the action is queued and it is executed when timeout exhausted. If Chii pushed again before timeout Chii completes and Pon is canceled.

1-han constraint：
If checked ON, your hand is checked that its value is at least 1 han. Win call is ignored and other player is not notified about your win call if NG. For complete hand, depending on "Win call is cancelable" enters into "Wait!" state or proceed to "Win Claimed". If not "Yes"(similar to real mahjong), miss-pressing "Win" button is cancelable if "Win is cancelable" is set, otherwise it may be "Chmbo" of mistaken Ron. For Locally defined winning pattern, you can proceed Ron by Win-A button or Win anyway of Menu even when err by checking hand. When "Cancel" button appeared, use "Win anyway" also for 2nd Win call for the local hand.
Evaluate Point By APP.：
"Yes":Evaluate Point(Fu and Han) by this APP. "No":The game is more interesting like real game.
If you do not agree the result, you can modify it on the dialog,
which allows to manage irregular Local Winning Hand.
Confirm Tenpai：
"No":Without checking Ron-able pattern, discards selected(green framed) tile. It may cause chombo of "NoTen Riich".

SuspendOption : "Request suspend" and "Cancel suspend" is menu items of during game. By "Request suspend" menu item, "SuspendGame Claim" dialog is shown at end of the hand on server. Select "Suspend", "Gameover" or "Continue". Default is defined in this topic and can be change by server on the dialog. Penalty can not be changed on the dialog while gaming. Getting agreements from all clients is required to proceed SuspendGame.
In case of ConnectionFailure, "ConnectionFailure Notification" dialog will appear. You can try reconnection. If failed to reconnect, select "Suspend"/"Gameover" with game status accomplished up to end of the previous hand.

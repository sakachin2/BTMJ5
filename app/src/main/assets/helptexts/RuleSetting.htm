Button Yaku : open child dialog related Yaku(winning pattern) such as "Kuitan", "Yakuman".
Button Oper : open child dialog related Operation such as "DelayTime", "Check winning hand".

ID : Identify the rule file, fill it such as Kansai, Tokyo etc to use rule file differently depending players group.
Use Save/Load button when updated.
On the right of ID, Update is displayed. This is key of rule synchronization. Unmatch of this among players prevents start of game even if its contents are same. When changed on server Send to all clients, game can be started when received Agree from all clients. If you want to use rule file on a client, the client "Load" the file then "Send" to server. Dialog will open on the server, which will be sent again to the other clients to synchronize.
OK : Update is saved as "Current" and stay activated.
Update is saved by "Send" button also. "Current" is kept in internal folder and removed by re-installation. Cancel : Updates are discarded.

InitialScore : TopPrize(Top players gain)=(30000-InitialScore)*4
Noten:3000: penalty of Noten at ExhaustiveDraw. Noten players pay totally 3000, Tenpai player gain 3000. e.g. 1 Noten pay 3000, 3 Tenpai gain 1000 each.
Counter:value of a counter-stick. Counter-stick(s) are shown on board when the dealer continues to stay as dealer. Winner gains additional point of (value of counter-stick)*(counter of counter-stick).

PlacePrize : payment by Placing(order).
e.g. when 5-10, Last player pays 10,000 to top player, 3rd player pays 5,000 to 2nd player.
HeadBump:The player closest to the starting dealer in counterclockwise sequence get higher placement. Divide(cut100):round down by 100.

Fraction : rounding under 100 points.
Negative point are rounded as following.
　4Down 5Up　: -100 ～ -500 ===> 0,　-600 ～ -900 ===> -1
　5Down 6Up　: -100 ～ -400 ===> 0,　-500 ～ -900 ===> -1
　Round-Up 　: -100 ～ -900 ===> 0
　Round-Down : -100 ～ -900 ===> 0
Top place player's final score is adjusted to keep zero-sum.

Dora： "End-of-Hand" dialog shows Dora indicator. Player has to count up han according this setting.

Tobi(Bankrupt)
Pay for negative score：Gameover after paying fully to winner.
return after placing：player pay once fully, after player's place is fixed by the final score, bankrupted player collect up to score=0 from the player levied.
Limit winner's point : winner's point is limited to the payer having now. Player's place is determined by the score.
If "Double-Ron":Yes, discarder pays to multiple winner according counterclockwise sequence starting from the winner closest to the discarder. Tobi prize is given to the player when bankrupt occurred in that sequence. In case of "NagashiMangan" starting collector is the closest to the dealer.

Robot：
Select Novice or Docile(Simply discard the taken avoiding Pao and Open Reach).
Smart robot issues Ron at the yellow period(for Pon/Kan) after Red(for Ron) time elapsed to give way to human winner.
When Robot(s) are included, robot's debts are responsible to human players if not playing alone.
Divide : divide equally robot's debts total, adjusting fraction by top place player's final score for zero-sum.
In proportion by score : divide proportionally as last place human player's pay is 0, divide proportionally.
    e.g score: +60000(top), +10000, -30000(last), -40000(robot)
       6-(-3)==>11, 1-(-3)==>4, -3-(-3)==>0  total=15 is divided.
       11/15*(-4)=-29300 (RoundDown by 100), 4/15*(-4)=-10600, 0/15*(-4)=0
       zero-sum adjusted -29400, -10600, 0, +40000 is added to score
       finally +30600, -400, -30000, 0(Robot)

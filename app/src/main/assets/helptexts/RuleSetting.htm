Button Yaku : open child dialog related Yaku(winning pattern) such as "Kuitan", "Yakuman".
Button Oper : open child dialog related Operation such as "DelayTime", "Check winning hand".

ID : Identify the rule file, fill it such as Kansai, Tokyo etc to use rule file differently depending players group.
Use Save/Load button when updated.
On the right of ID, timestamp of Update is displayed. This is key of rule synchronization. Unmatch of this among players prevents start of game even if its contents are same. When changed on server Send to all clients. When received dialog opens and you can identify items different with current setting by background color. Game can be started when received Agree from all clients. If you want to use rule file on a client, the client "Load" the file then "Send" to server. Dialog will open on the server, which will be sent again to the other clients to synchronize.
OK : Update is saved as "Current" and stay activated.
Update is saved by "Send" button also. "Current" is kept in internal folder and removed by re-installation. Cancel : Updates are discarded.

Rounds :
東風戦:4 rounds, East(東) wind only game.
東東戦:8 rounds, East only game.
東南戦:4 rounds by East(東) then 4 rounds by South(南).
東北戦:4 rounds by East(東) then 4 rounds by North(北).
一荘戦:whole total 16 rounds by East(東), South(南), West(西) then North(北).
InitialScore : TopPrize(Top player's additional gain)=(30000-InitialScore)*4
Noten:3000: penalty of Noten at ExhaustiveDraw. Noten players pay totally 3000, Tenpai player gain 3000. e.g. 1 Noten pay 3000, 3 Tenpai gain 1000 each.
Counter:value of a counter-stick. Counter-stick(s) are shown on board when the dealer continues to stay as dealer. Winner gains additional points of (value of counter-stick)*(counter of counter-stick).

PlacePrize : payment by Placing(order).
e.g. when 5-10, Last player pays 10,000 to top player, 3rd player pays 5,000 to 2nd player.
HeadBump:The player closest to the starting dealer in counterclockwise sequence get higher placement. Divide(cut100):round down by 100.

Fraction : rounding under 100 points.
Negative point are rounded as following.
　4Down 5Up　: -100 ～ -500 ===> 0,　-600 ～ -900 ===> -1
　5Down 6Up　: -100 ～ -400 ===> 0,　-500 ～ -900 ===> -1
　Round-Up 　: -100 ～ -900 ===> 0
　Round-Down : -100 ～ -900 ===> 0
Top place player's final score is adjusted to keep zero-sum.

Dora： Dora tile has value of 1 han each in your Hand.
A Dora is randomly decided and shown on Stock at start of each round, and increase at Kan(declare quad). "End-of-Hand" dialog shows Dora indicator.
Kan-Dora is not opened for Chankan(Win called for the tile of Add-Kan).
Hidden : Tile underneath the Dora tile opend on Stock is hidden Dora. It is shown only when the player called Riichi won.
Itself : Dora is the same tile as opend.
NextTo : Dora is next tile of the opend. Number tile:1-9-->2-0, dragon:WGR-->GRW, wind:ESWN-->SWNE
Imm. means open Kan-Dora at additional Draw for Kan, else open at after discarded.
RedDora： Half of number-5 tiles of each suit is colored red as additional Dora.(total 6 tiles more Dora)

Wareme：
Wareme(Japanese:split point) player is decided by the role of Dice the dealer casted at start of each round.
dice spot=2-->南 is Wareme, 3-->西, 4-->北, 5-->東, 6-->南, ...

Tobi : (Japanese: Fly away, Bankrupt)
Pay for negative score：Gameover after paying fully to winner.
return after placing：player pay once fully, after player's place is fixed by the final score, bankrupted player collect up to score=0 from the player levied.
Limit winner's point : winner's point is limited to the payer having now. Player's place is determined by the score.
If "Double-Ron":Yes, discarder pays to multiple winner according counterclockwise sequence starting from the winner closest to the discarder. Tobi prize is given to the player when bankrupt occurred in that sequence. In case of "NagashiMangan" starting collector is the closest to the dealer.

Yakitori：Japanese:YAKI(grill)-TORI(bird, chicken)
This mark indicate the player won never once in the game. At End-Of-Game the player holding the mark pays award. This point is payed after player placing determined.
Divide:Marked player pays the point and Not-maked players get equally divided point.
To All:Marked player pays the point to all Not-maked players.

Call the same Meld： Limitation to discarding after calling Pon/Chii.
No if the same tile：e.g) Calling Chii for number-1 to make meld 1-2-3 and discarding number-1 is not allowed, but discarding number-4 is allowed. Likewise, calling Pon a tile and discarding the same tile is not allowed.
No：Both is not allowed.
OK fully：Both is allowed.

Abortive Draw：
4Riichi：(all player called Riich)
Dialog popup automatically if no one called Ron for the discarded tile after elapsed some seconds.
4Winds：(The first 4 discarded tiles are all one of 東南西北)
Dialog popup automatically. If conceiled Kan was called in the first 4 Discardit, it is not a case of 4Winds. But you can request to open the dialog from Menu.
3Win：(3 players called Win at same time)
"AbortiveDraw" dialog popups when "Fix" was pushed on "End-Of-Hand" dialog. Effective even if "Double-Ron" is "No".
e.g) 4Riichi:When 4 player called Riichi any player can call "Abortive Draw" from Menu. And the next round to be started with dealer stay or not. If no one called "Abortive Draw" the round continues.

Double-Ron：
No:HeadBump:Only the player closest to the discarder in counterclockwise sequence is granted the win.
YesAllow all player's win. Counter stick is by HeadBump rule. Yes also...pay also Counter-Stick to all winners.
Robot's Ron is ignored regardless this option if human player called Ron in Red period(for Ron) except when playing alone mode.

Robot：
Name of robot is randomly selected from "S それがし", "A あきらくん", "T てるちゃん", "M まあちゃん", "K けんちゃん".
Japanese pronunciation are Soregasi(means Me or Myself), AKIRA-kun, TERU-chan, MAH-chan and KEN-chan each, and are my childhood frends.
Smart robot issues Ron at just before entering the yellow period(for Pon/Kan) and gives way to human winner if exists. But in Play Alone mode, human player has no priority and Double-Ron rule is applied.
When the game includes Robot(s) except playing alone mode, robot's debts are responsible to human players.
Divide : divide equally robot's debts total, adjusting fraction by top place player's final score for zero-sum.
In proportion by score : divide proportionally as last place human player's pay is 0, divide proportionally.
    e.g score: +60000(top), +10000, -30000(last), -40000(robot)
       6-(-3)==>11, 1-(-3)==>4, -3-(-3)==>0  total=15 is divided.
       11/15*(-4)=-29300 (RoundDown by 100), 4/15*(-4)=-10600, 0/15*(-4)=0
       zero-sum adjusted -29400, -10600, 0, +40000 is added to score
       finally +30600, -400, -30000, 0(Robot)
